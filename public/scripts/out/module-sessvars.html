Module: sessvars

session variables without using cookies. It will let you store 2 MB of data, with much less hassle than a cookie based solution. The only property of sessvars you shouldn't tamper with is $, because it contains a number of helpful methods.

Example

You will be able to access the object sessvars.myObj on all the other pages the user navigates to during the same session

sessvars.myObj = {name: "Thomas", age: 35}

(inner) flush(x) → {Boolean}

Explicitly saves the current state of sessvars so that all data survives when you go to the next page. This should rarely be needed, as it is normally done automatically when the unload event fires.
